# Phase 5 設計書 — 副業モード（給与あり）の税試算

- 作成日: 2026-06-10
- 対象: 会社員（本業給与）＋フリーランス（副業）のユーザー
- 親設計書: `2026-06-04-phase4-payment-projection-withholding-design.md`
- ステータス: 設計確定（実装プラン作成へ）

## 1. 背景・目的

現在の税試算は「専業フリーランス（事業所得のみ・国保＋国民年金）」を前提にしている。
会社員＋副業の場合、①社会保険は勤務先負担（国保・国民年金は不要）、②副業の税は給与に上乗せした限界税率で効く、という前提が異なり、**現状モデルは取り置き過多**になる。

本フェーズでは「就業形態」設定を追加し、副業モードでは実態に合った追加税（所得税＋住民税のみ）と取り置き目安を算出する。

## 2. スコープ

### やること
- `tax_settings` に `employment_type`（専業 / 給与あり）と `salary_income`（給与年収見込み）を追加。
- 給与ありモードの計算：
  - 給与所得控除テーブル（純関数・TDD）で給与所得を算出。
  - 合計課税所得 = 給与所得 ＋ 事業所得 から限界税率（差分）で**副業分の追加所得税**を算出。
  - **国保・国民年金は除外**（社保控除として給与分の厚生年金＋健保を概算計上）。
  - 取り置き = 追加所得税 ＋ 追加住民税（事業所得×10%）－ 源泉。
- 専業モードは現行通り不変（後方互換）。

### やらないこと
- 配偶者控除・扶養控除などの人的控除（`other_deductions` の手入力で代替）。
- 副業所得が20万円以下の申告不要判定（概算ツールのため）。
- 住民税の特別徴収と普通徴収の分離。

## 3. データモデル

### 3.1 tax_settings への列追加（マイグレーション `0006_phase5.sql`）

```sql
alter table tax_settings
  add column employment_type text not null default 'freelance'
    check (employment_type in ('freelance', 'salaried')),
  add column salary_income   numeric not null default 0;
```

- `employment_type`: 'freelance'（専業・現行と同じ）/ 'salaried'（給与あり・副業）。
- `salary_income`: 給与収入の年額見込み（税込み額面）。目安：昨年の源泉徴収票「支払金額」、なければ月給×12＋賞与。'freelance' モードでは無視。
- 既存行は default で 'freelance' になるため後方互換。

### 3.2 TaxSettings 型（lib/types.ts）
```ts
employment_type: 'freelance' | 'salaried'
salary_income: number
```

## 4. 純関数ロジック（TDD）

### 4.1 lib/salary.ts — 給与所得控除

```
calcSalaryIncome(salaryRevenue: number): number
```

2026年時点の給与所得控除テーブル（改正時に更新）：

| 給与収入 | 控除額 |
|---|---|
| 〜1,625,000 | 550,000 |
| 〜1,800,000 | 収入 × 40% − 100,000 |
| 〜3,600,000 | 収入 × 30% + 80,000 |
| 〜6,600,000 | 収入 × 20% + 440,000 |
| 〜8,500,000 | 収入 × 10% + 1,100,000 |
| 8,500,000超 | 1,950,000（上限） |

給与所得 = max(給与収入 − 給与所得控除, 0)

テストケース（最低限）：
- 0 → 0
- 1,000,000 → 450,000（控除 550,000）
- 3,000,000 → 2,020,000（控除 980,000）
- 5,000,000 → 3,560,000（控除 1,440,000）
- 10,000,000 → 8,050,000（控除 1,950,000）
- 境界値: 1,625,000 → 1,075,000（控除 550,000）、1,625,001 → 550,000×40%−100,000 分岐確認

### 4.2 lib/tax.ts 拡張 — 給与ありモード

`TaxInput` に追加：
```ts
employmentType?: 'freelance' | 'salaried'  // default 'freelance'
salaryIncome?: number                       // default 0
```

`TaxResult` に追加（給与ありモードのみ意味を持つ；専業モードは 0）：
```ts
salaryIncome: number          // 入力された給与収入
salaryDeduction: number       // 給与所得控除額
salaryEarnings: number        // 給与所得（収入 − 控除）
```

**給与ありモードの計算ステップ：**

```
給与所得 = calcSalaryIncome(salaryIncome)

// 勤務先の社会保険料を概算（控除として使用）
// 厚生年金: salaryIncome × 9.15%、健保: salaryIncome × 5%（概算）
給与社保概算 = round(salaryIncome × 0.1415)

// 事業所得（従来通り）
事業所得 = max(売上 − 経費 − 青色控除, 0)

// 合計課税所得（所得税）
課税所得_合計(所得税) = max(給与所得 + 事業所得 − 給与社保概算 − 基礎控除(所得税) − other_deductions, 0)
課税所得_給与のみ(所得税) = max(給与所得 − 給与社保概算 − 基礎控除(所得税), 0)

// 副業分の追加所得税 = 合算税額 − 給与のみ税額
追加所得税 = round(progressiveIncomeTax(課税所得_合計) × 1.021) 
           − round(progressiveIncomeTax(課税所得_給与のみ) × 1.021)
追加所得税 = max(追加所得税, 0)

// 副業分の追加住民税（所得割のみ。均等割は勤務先で徴収済）
追加住民税 = round(事業所得 × resident_tax_rate)
           // resident_tax_fixed は給与側で既に払われているため加算しない

// 社会保険：給与ありモードでは国保・年金は 0
nationalPension = 0
healthInsurance = 0
socialInsuranceDeduction = 0

// 合計・手取り
totalTaxAndInsurance = 追加所得税 + 追加住民税
netIncome = 売上 − 経費 − totalTaxAndInsurance
```

**reserve（取り置き）の補正：**
- `reserveBase = max(totalTaxAndInsurance − withholding, 0)`（源泉控除、従来通り）

**ゲート：事業所得 = 0 のとき全て 0（従来通り）**

**テストケース（最低限）：**
- 給与 500万・副業売上 100万・経費 20万・青色65万（事業所得 15万）→ 追加税が少額になることを確認。
- 給与 800万・副業売上 600万・経費 100万 → 高い限界税率（33%〜）での追加税。
- 給与 0・専業モード → Phase 3/4 と同一（後方互換）。
- 事業所得 0 → 全て 0。

## 7. 設計判断メモ

- 給与社保概算（salaryIncome × 14.15%）はあくまで概算。標準報酬月額の上限丸め等は無視する（個人差大・概算ツールの範囲内）。
- 住民税均等割は給与の特別徴収で支払済みのため副業側には加算しない。
- 本業給与の源泉徴収（年末調整済み税額）は `calculateTax` の対象外。副業分の追加税のみ算出する。
- `progressiveIncomeTax` は既に「生値（端数処理前）」を返す設計になっており、限界税率の差分計算でも精度を保てる（二重丸め問題を回避済み）。
